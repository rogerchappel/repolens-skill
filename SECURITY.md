# Security Policy

## Supported Versions

This project is a v0.1.0 public skill package. Security fixes are made on the
default branch until a stable release line exists.

## Reporting a Vulnerability

Please report suspected vulnerabilities through GitHub private vulnerability
reporting when available, or by opening a minimal public issue that avoids
sharing private repository contents, credentials, tokens, customer data, or
unpublished code.

Include the affected version or commit, reproduction steps using the checked-in
fixtures, and the expected impact. Do not include proprietary source snapshots
or private repository summaries in public reports.

## Data Handling

The skill is designed for local repository review briefs. It should summarize
only the repository snapshot supplied by the user and must not upload source
content, write to external services, or perform approval-requiring actions.
